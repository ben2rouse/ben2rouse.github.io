Assignment Hub Dashboard

This dashboard is the final result of a university capstone project I had the privilege of working on with Jeffrey Filiberto


This project was aimed at utilizing the OpenAI API to process unstructured school syllabi with the goal of creating a structured assignment database.


This report is the end product on my end of the project, where I visualized the student workload over time to study peak busy times for students in order for on-campus planners to incorporate student workload into their calculations when planning student events.

Alumni Survey Report

The report helped to explain alumni satisfaction with their experience, their motivators for attending, and work details upon leaving the university.

This report page shows how well prepared by the university alumni felt in a range of categories.

This report page shows whether alumni would attend or take the same major again. I included a comparison visual that can be opened/closed, to compare between demographics.

This page explores the motivators for university attendance (with a comparison chart to compare between demographics).

This page shows which industries alumni are working in, how prepared they felt, and the distribution of salaries in their current work.

Delaware Cost Study

This report was transitioned from an existing Excel spreadsheet to Power BI in order to automate the process and decrease setup time every semester.


The report allows the university to compare departmental costs to other universities of similiar size and focus.

This is the landing page which explains the purpose of the report as well as guidelines in interpeting the comparisons.

This page shows a year by year analysis of costs per department. The table uses measures to dynamically calculate the colors whcih represents how well the dept. compares to the averages.

This is a snapshot of the tooltip which allows a user to take a deeper look at the quartiles that we are comparing against.

This page focuses on a single department and allows us to see more comparison detail as well as various attributes of the department.

This page allows the user to have a longitudinal view of all or specific departments and how they have compared to the quartile metrics over time.

Senior Exit Survey Report

Similiar to the Alumni Survey report, this Senior Exist Survey helps administration assess how students felt about their college experience, looking across various categories.

This is the main page for the report. It presents each page of the report and a dropdown for each with an explaination of the page. It also serves as the navigation page.

Student reponses are (on a likert scale) are ranked and then we present the number of responses, the mean, and the standard deviation for each question. This can be compared to the university values and we can see the number of comments specific to that question.

When hovering over a specific question, a longitudinal graph is shown to visualize how the departmental mean has compared to the university values.

Each question on the previous page can be used to drill through to the comments that are tied to specific questions. This page shows the comments, what the student responded to the question, and their department.

This page shows a more detailed view of the likert options for each question, allowing the user to see the distribution of likert options to each question.

At the end of the Exit Survey, more general comments are allowed. This page allows users to see these comments for each department and year.

Users can click on a specific department (or a specific department within a specific year) and the comments filter accordingly. The "Other Department" column in the right table shows whether the student has a degree in a secondary department.

Gen Ed Assessment Report

The university assesses the effectiveness of their general education courses using questionnaires, rubrics, and graded items. This report is where users can see how well each section or course is doing in their respective Gen Ed area.


This report is under development currently, but here I show some of the process for collecting, correcting, and working with the data to show the results.

This is one of many pages of the report that shows how much of the course population Does Not Meet, Meets, or Exceeds the expectations for the Gen Ed area.

We utilize a tool called VEERA Construct to perform the ETL in order to prepare the data to complete our assessment. This is a snapshot of pulling just the questionnaire data from the databases, merging that data with other data, and then transforming for export.

This is an example a query in Excel's Power Query to process the raw data coming from VEERA to provide assessments for each student response.

With individual response assessments created, I pull the data into Power BI. I pull some data from our processed assessment data, but some tables are pull straight from the database.

Here you see all of the relationships that I have created between tables to allow our report to filter and connect data correctly.

With all of the prep work in order, I created this page to display the assessment of whether the sections/courses are meeting expectations. This uses measures to dynamically calculate the population assessments from the individual responses.

Here is an expanded view of the previous table, showing the assessments in more details by section and term.
