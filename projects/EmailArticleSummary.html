Email/Article Summary Digest Automation

Automated daily tech digest and article summaries delivered to your inbox

Project Overview

This project automates the process of retrieving, summarizing, and emailing daily tech digests and curated articles. It integrates with Gmail, Raindrop.io, and OpenAI’s API to deliver a categorized summary email every morning. The workflow is fully automated and cron-ready, making it a powerful tool for staying up-to-date with the latest in tech, AI, data, and more.

View on GitHub

How It Works

  • Fetches TLDR newsletter emails from Gmail and extracts categorized digests
  • Summarizes each digest and saved article using OpenAI GPT models
  • Fetches and processes saved articles from Raindrop.io
  • Composes a polished HTML email with summaries and clickable article links
  • Tracks OpenAI API token usage and cost per run
  • Fully automated daily workflow via cron job
  • Robust error handling and logging
Section Summaries: Each email includes a summary for every category (like AI, Data, Web Dev), with an overall summary and individual article summaries—each title is a clickable link.
Article Summaries: Article summaries are formatted with the article title as a link, followed by a concise summary generated by OpenAI.
API Usage & Cost: At the end of each email, the workflow reports OpenAI API usage, model, token count, and cost for full transparency.

Project Structure

  • EmailDigest.py — Fetches TLDR emails and saves category digests as .txt files in dated folders
  • DigestToSummary.py — Summarizes each digest file and saves results in a Summary subfolder
  • FetchRaindropArticles.py — Fetches untagged articles from Raindrop.io, stores all data in articles.json
  • SummarizeArticles.py — Summarizes articles, updates articles.json, writes token usage to token_usage.txt
  • SummaryToEmail.py — Builds and sends HTML summary email, includes token/cost info
  • RunAll.py — Master script to run all steps in sequence
  • PROJECT_OVERVIEW.md — Project documentation and workflow overview
  • README.md — Setup instructions, features, and usage
  • .env — Stores credentials and model selection
  • requirements.txt — Python dependencies
  • digests-YYYY-MM-DD/ — Dated folder for each daily run, with summaries, articles, and token usage
